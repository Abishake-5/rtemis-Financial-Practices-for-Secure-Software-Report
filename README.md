# CS 305 Portfolio Artifact: Artemis Financial Practices for Secure Software


## Artifact Included
Completed Artemis Financial Practices for Secure Software Report

## Reflection

Artemis Financial was a client that needed stronger software security to help protect sensitive financial data. The company wanted vulnerabilities identified and security improvements added to reduce risk and improve secure communication. One major issue addressed in this project was improving data protection in transit and verifying data integrity.

One thing I did well in this project was identifying security concerns and applying solutions such as SHA-256 hashing and HTTPS communication. This project showed me why secure coding is important because weak security can lead to data breaches, system failures, and loss of trust. Software security adds value to a company by protecting customer information, reducing risk, and supporting reliable operations.

One challenging part of the project was understanding dependency-check results and reviewing vulnerabilities connected to third-party libraries. At the same time, this was helpful because it improved my ability to analyze risks and prioritize mitigation strategies.

I increased layers of security by implementing SHA-256 for integrity verification, configuring HTTPS with SSL certificates, and using OWASP Dependency-Check to review dependencies. In the future, I would continue using vulnerability scanning tools, static analysis tools, CVE databases, and secure coding standards to assess risks and choose mitigation techniques.

To make certain the software remained functional and secure, I tested the application after refactoring and reran security scans to verify no new vulnerabilities were introduced. This helped confirm that the security changes did not break functionality.

Resources and tools I used that will help me in future work include OWASP Dependency-Check, Maven, secure coding practices, CVE research, and documentation review.

For future employers, I could show this project as evidence of my ability to assess software risks, apply secure coding practices, improve security controls, and validate software after changes. This artifact demonstrates both technical knowledge and practical problem-solving skills.
